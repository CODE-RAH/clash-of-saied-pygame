# 📝 درباره پروژه

این پروژه یک **بازی اکشن دو بعدی (2D Side-Scroller)** است که با زبان **Python** و کتابخانه قدرتمند **Pygame** توسعه یافته است.

در این بازی، شما در نقش کاراکتر اصلی (سعید) قرار می‌گیرید که باید در برابر موج‌های دشمنان ایستادگی کرده، از پرش‌های چابکانه استفاده کند و با شلیک دقیق، دشمنان را نابود کنید. هدف، بقا در میدان نبرد و کسب بیشترین امتیاز (Kills) است.

---

# ⚡ ویژگی‌های کلیدی بازی

- **گرافیک Sprite-Based**  
  استفاده از تصاویر سفارشی برای کاراکتر، دشمنان، گلوله‌ها و قلب‌های سلامت (HUD) با پشتیبانی از transparancy.

- **سیستم فیزیک پرش (Jump Physics)**  
  شبیه‌سازی واقع‌گرایانه گرانش و سرعت عمودی برای حرکات روان کاراکتر روی پلتفرم.

- **هوش مصنوعی دشمنان (AI)**  
  دشمنان به صورت خودکار به سمت بازیکن حرکت کرده و به صورت تصادفی شلیک می‌کنند (Random Shooting).

- **سیستم سلامت (Health System)**  
  نمایش گرافیکی قلب‌های قرمز به صورت دو ردیفی (UI Dynamic) و کاهش جان در اثر برخورد یا اصابت گلوله.

- **گیم‌پلی اکشن**  
  قابلیت شلیک همزمان گلوله توسط بازیکن و دشمنان با مدیریت collision detection دقیق.

---

# 🛠️ نحوه کار با بازی و معماری کد

کدهای این پروژه به صورت ماژولار و شی‌گرا طراحی شده‌اند:

1. **راه‌اندازی اولیه**  
   با اجرای فایل اصلی، Pygame init شده و صفحه بازی با ابعاد $800 \times 400$ پیکسل ایجاد می‌شود.

2. **حلقه اصلی بازی (Game Loop)**  
   با نرخ $55$ فریم بر ثانیه (FPS) اجرا شده و رویدادهای صفحه کلید (KeyEvents) را برای حرکت و شلیک پردازش می‌کند.

3. **مدیریت موجودیت‌ها (Entity Management)**  
   - لیست `enemies` برای ذخیره دشمنان به صورت `pygame.Rect`
   - لیست‌های `player_bullets` و `enemy_bullets` برای مدیریت پروژکتایل‌ها
   - بررسی برخورد (Collision Detection) بین مستطیل‌ها (Rect collision)

4. **سیستم امتیازدهی**  
   متغیر `kills` شمارنده دشمنان نابود شده و `player_health` وضعیت سلامت (حداکثر 10 قلب) را مدیریت می‌کند.

## 🖼️ مرحله سوم: آماده‌سازی تصاویر (Assets)

این بازی برای اجرا نیاز به فایل‌های تصویری زیر در همان پوشه اصلی دارد:

- `enemy.png` (تصویر دشمن - سایز پیشنهادی: $100 \times 100$)
- `bullet.png` (تصویر گلوله بازیکن - سایز پیشنهادی: $10 \times 5$)
- `enemy_bullet.png` (تصویر گلوله دشمن - سایز پیشنهادی: $10 \times 5$)

- `grass.png` (تصویر زمین/چمن - سایز پیشنهادی: عرض $800$ پیکسل)

> 💡 **نکته:** اگر تصاویر را ندارید، می‌توانید فایل‌های PNG ساده با رنگ‌های مختلف ایجاد کنید یا از تصاویر placeholder استفاده نمایید.
